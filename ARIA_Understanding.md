## 1. The Core Vision — What ARIA Actually Is

ARIA is not a chatbot. It is not a content tool. It is not a dashboard. ARIA is a **one-input marketing operating system** that replaces every marketing tool a business currently pays for, and executes the work automatically.

The product thesis is simple: people do not come to ARIA for a "campaign." They come because they need something right now — a video, a landing page, an email sequence, a competitor analysis. ARIA reads that intent, routes it to the correct agents, and delivers a complete, professional output. No menus. No forms. No navigation. One input box.

> "ARIA is the product. Everything else is infrastructure running underneath."

The competitive window is real and time-limited. Mega AI raised $11.5M from a16z on March 10, 2026 at $299/month with humans in the loop. ARIA delivers the same capability — fully automated — at $49/month. That gap will not last 12–18 months.

## 2. How ARIA Works — The Three Checks

Before building anything, ARIA runs three checks on every single request:

| Check | What ARIA Does |
|---|---|
| **Who are you?** | Loads Brand Kit — tone, colors, audience, competitors. If none exists, asks ONE question to begin building it. |
| **What have you built?** | Checks folder history. References past campaigns to inform the current build. Never starts from zero for returning users. |
| **What do you need now?** | Reads intent from the message. Routes to the correct module. Builds immediately. |

---

## 3. The Three Entry Points

ARIA detects which mode the user is in from their first message. No selection required.

**Entry Point 1 — Full Campaign Builder.** Triggered when the user wants to launch something new or needs the full marketing stack. The flow is: (1) user types one sentence → ARIA searches the web for the existing brand → returns a 3–4 bullet strategy summary → "Build Campaign" button appears; (2) a dynamic checklist pre-populated from domain intelligence appears — not hardcoded, different every time; (3) a progress screen shows each agent completing by name, then the full campaign folder opens automatically.

**Entry Point 2 — Existing Brand.** Triggered when the user has a website or existing brand. ARIA reads the brand from the URL or upload, loads the Brand Kit automatically, shows only what is missing, and never rebuilds what already exists.

**Entry Point 3 — Specific Task.** Triggered when the user needs one thing fast. Zero questions. Zero checklist. Immediate execution. Saves to a folder automatically. Option to expand into a full campaign after delivery.

### 4.4 AEO — Answer Engine Optimization (NEW in v2)

This is a **first-mover feature** that no other marketing platform currently offers as an automated module. AEO is the discipline of getting a brand recommended by AI search engines (ChatGPT, Perplexity, Google AI Overviews, Gemini). Traditional SEO ranks pages. AEO gets brands named as the answer.

The full AEO loop ARIA runs: (1) AI Presence Audit — queries ChatGPT, Perplexity, and Google AI Overviews with the top questions in the user's category; (2) Gap Identification — shows which questions the brand should answer but doesn't; (3) Content Build — writes the full question cluster (20–30 pieces), entity page, and data asset with schema markup; (4) Distribution — SocialAgent and EmailAgent push the content; (5) Re-audit — 30 days later, re-runs the audit and shows if the brand now appears in AI recommendations.

**AEO Deliverables per campaign folder:**
- AI Presence Audit (which AI engines recommend you, which don't)
- Entity Page (authoritative brand brief with Organization schema)
- Question Cluster (20 answer-ready content pieces)
- Original Data Asset (one research post designed to generate citations)
- Schema Markup Package (structured data for every page type, paste-ready)
- PR Outreach Brief (10–15 target publications with pitch angles)
- 30-Day Re-Audit (automated re-check after content is indexed)

## 13. UI Directives — Confirmed Changes

### 13.1 Remove All LLM Branding from the UI
The current build shows "Claude Sonnet 4.5" in the ARIA chat header and sidebar. **Users do not need to know what AI model is running underneath.** This is backend infrastructure. Remove all references to Claude, Sonnet, Haiku, GPT, or any specific model name from every user-facing surface. The AI is ARIA. That is the only brand.

### 13.2 Sidebar Bottom Bar — User Profile
The current sidebar bottom bar shows the user's full name and email address visibly. This is not standard practice and wastes space. Replace with a compact icon-only approach: a small avatar/icon that, when clicked, opens a popover or dropdown showing the name, email, and account options (Settings, Billing, Logout). Follow the pattern used by tools like Notion, Linear, and Vercel — the identity is accessible but not permanently displayed.

### 13.3 Campaign Folder as Primary UI
The chat thread is not the product. The campaign folder is the product. Pass 1 replaces the current chat-only interface with the three-step builder that terminates in a campaign folder view. The chat bar moves to the bottom of the folder as a refinement tool — not the primary interface.
